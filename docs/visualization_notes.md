# Visualization Notes

## Purpose

This file records the visualization outputs used in the LOG-LIO reproduction.

The project uses two types of visualization:

1. offline trajectory and error plots generated from saved result files;
2. RViz screenshots generated from a ROS Noetic Docker container with WSLg display support.

## RViz Visualization

RViz was launched in a ROS Noetic Docker container using WSLg display support.

The main RViz visualization was performed on the M2DGR `room_01` sequence.

Displayed topics:

| Topic | Meaning |
|---|---|
| `/Laser_map` | Local point-cloud map generated by LOG-LIO |
| `/path` | Estimated trajectory path |

RViz screenshot files:

| Figure | Path |
|---|---|
| RViz map screenshot 1 | `results/figures/room_01_rviz_map_1.png` |
| RViz map screenshot 2 | `results/figures/room_01_rviz_map_2.png` |

These screenshots show that LOG-LIO was not only evaluated from saved trajectory files, but also visualized in a ROS/RViz environment.

## Notes

The RViz path is an estimated odometry trajectory, not a planned navigation route. LOG-LIO estimates where the sensor platform moved; it does not perform path planning or autonomous driving control.

The RViz screenshots are used as qualitative evidence only. Quantitative evaluation is reported separately in `docs/evaluation_results.md`.
