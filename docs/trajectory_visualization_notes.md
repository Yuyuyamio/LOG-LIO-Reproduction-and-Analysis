# Trajectory Visualization Notes: M2DGR door_02

## Purpose

This step visualizes the trajectory generated by LOG-LIO on the M2DGR `door_02` sequence. The goal is not to claim final quantitative accuracy yet, but to check whether the estimated trajectory is continuous, non-empty, and physically meaningful.

## Input

The input trajectory file is:

results/door_02_run1/target_path.txt

Each row follows the common TUM-style trajectory format:

timestamp x y z qx qy qz qw

## Generated Figures

Two SVG figures were generated:

results/door_02_run1/trajectory_xy.svg

results/door_02_run1/trajectory_z_time.svg

## Initial Observation

The XY trajectory figure shows the horizontal movement pattern estimated by LOG-LIO. The start and end points are marked separately, making it easier to check whether the system produced a continuous motion trajectory rather than a static output.

The z-time figure shows how the vertical position changes during the sequence. This is useful for checking whether the estimated height remains within a reasonable range and whether there are obvious jumps.

## Current Conclusion

The visualization confirms that LOG-LIO produced a continuous trajectory for the M2DGR `door_02` sequence. Together with the previous trajectory sanity check, this supports that the reproduction has passed the basic run-and-output verification stage.
